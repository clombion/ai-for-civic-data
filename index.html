The 5 principles

Every case study on this site applies the same five principles. They are not a checklist to follow after the fact — they are decisions you make before you open a chat window or run a pipeline. Skipping any one of them is where things go wrong.

Applied to every case study
01
Objective
Define what success looks like in concrete, checkable terms — before touching AI. Not "summarise this document" but "extract all commitments naming a specific actor, obligation, and timeframe — with the paragraph reference for each."
02
Output shape
Specify the structure and fields of the result before you ask for it. A table with fixed columns, a JSON schema, a numbered list with named fields. Without a defined shape, AI produces narrative that is hard to verify and impossible to compare across documents.
03
Atomic steps
Break the task into explicit, ordered sub-tasks and ask AI to complete them one at a time. Do not ask AI to "do everything at once" — each step should have a defined input, a defined output, and a verification point before the next step begins.
04
Reasoning trace
Require AI to show its assumptions and interpretive choices at each step — not just its output. Without a trace, you see only the conclusion. A confidence field, a note field, or inline code comments all count. If AI can't show its reasoning, you can't evaluate it.
05
Human review
Design how a human will verify the result before you run anything. Define which outputs require mandatory review, which require spot-checks, and which can be validated automatically. The review process is part of the work, not an afterthought.
Prompts and skill files

Every case study gives you something concrete to use. The prompt is a structured text template that applies the 5 principles to that specific task. The skill file (pipeline cases only) is a reusable instruction set for agentic AI tools — it automates the multi-step process so you don't have to run each step manually.

Choosing the right approach
Use the prompt if…
You want to run a one-off task, try the approach before committing to a setup, or work with a dataset small enough to handle in a single session. The prompt works in any AI chat interface with no installation required.
Use the skill file if…
You need to process a corpus of files, repeat the task regularly, or want an automated pipeline that handles batching, validation, and audit logging. Skill files require a tool that can run agentic AI workflows — Claude Code, Claude.ai Projects, or Gemini CLI.
Using a prompt in a chat window

The simplest way to use any case study. Works with Claude.ai, ChatGPT, Gemini, or any AI assistant.

1
Open a new conversation
Go to claude.ai (or your preferred AI assistant) and start a new chat. Each case study works best in a fresh conversation so the AI has no prior context to carry over.
2
Copy the prompt from the case study
On any case study page, click the Prompt tab, then the Copy button. The prompt is a structured template with [PLACEHOLDER] slots you will fill in.
3
Fill in the placeholders
Before sending, replace every [PLACEHOLDER] with your actual content — your document text, your specification, your questions. Do not skip placeholders or leave them blank. Each one corresponds to a decision the 5 principles require you to make.
4
Send and review the output
The prompt is designed to produce structured output you can verify. Check the Human review accordion on the case study page for what to look for. For long documents, run the prompt section by section and compile the results yourself.
Tip: Keep the case study page open alongside your chat. The principles accordion explains why each part of the prompt is structured the way it is — useful when the output surprises you.
Using a prompt as a custom chatbot

A custom chatbot lets you save the prompt as a persistent system configuration, so you don't have to paste it each time. Good for tasks you run regularly or want to share with a team.

1
Open Claude.ai Projects
In claude.ai, click Projects in the sidebar, then New project. Projects allow you to set a persistent system prompt that applies to every conversation in that project.
2
Paste the prompt into Project Instructions
In your new project, click Edit project instructions. Paste the full prompt from the case study. Fill in any placeholders that are fixed for your use (e.g. your extraction schema, your codebook). Leave placeholders that will vary per conversation.
3
Upload reference files
If your case study uses a schema, codebook, or alias table — upload those files to the project. Claude will reference them in every conversation without you needing to paste them each time.
4
Start a new conversation per document or task
Each new conversation in the project starts fresh with your system prompt active. Paste your document section, your dataset profile, or your search specification directly into the chat — the prompt structure handles the rest.
Also works with: The Anthropic API (set the prompt as the system parameter), or any platform that supports a system prompt field — including Poe, Dify, or Flowise.
Running a skill with Claude Code

Claude CodeClaude CodeAn agentic coding tool from Anthropic that runs in your terminal. It can read and write files, execute code, and run multi-step workflows — making it suitable for pipeline skill files that process batches of documents. is an agentic tool that runs in your terminal. It can process files, run scripts, and execute multi-step pipelines — ideal for skill files that handle a corpus of documents.

1
Install Claude Code
Claude Code requires Node.js. Install it via npm:
npm install -g @anthropic-ai/claude-code
Then authenticate: run claude and follow the login prompt. Full docs at docs.anthropic.com/claude-code.
2
Download the skill file
On the case study page, click the Skill tab, then Download skill file. Save the .md file into a skills/ folder in your project directory.
3
Set up your working directory
Create a folder for the task and add your input files. For example, for thematic analysis:
my-analysis/ skills/thematic-analysis-skill.md input/document-01.txt input/document-02.txt codebook.yaml
4
Run Claude Code with the skill
From your project directory, start Claude Code and reference the skill file:
cd my-analysis claude > Read skills/thematic-analysis-skill.md and follow it.
Claude Code will read the skill, ask any clarifying questions, then execute the pipeline — processing your input files, producing outputs, and logging its work.
Note: Skill files include built-in review gates. Claude Code will pause at mandatory human review points and wait for your confirmation before continuing.
Running a skill with Claude.ai

For users who prefer a browser interface rather than the terminal. Claude.ai Projects can run skill files when you upload them as project instructions alongside your input files.

1
Create a new Project in Claude.ai
At claude.ai, click Projects → New project. Give it a name matching the task (e.g. "Thematic Analysis — Budget Documents").
2
Upload the skill file as project instructions
Download the skill file from the case study. Open Edit project instructions and paste the full contents of the skill file. This sets the operating procedure for every conversation in this project.
3
Upload your reference files
Upload your codebook, schema, alias table, or any other reference file the skill requires. These will be available to Claude in every conversation within the project.
4
Start a conversation and paste your input
Start a new chat. Paste your input — a document section, a dataset profile, a specification. Claude will follow the skill's procedure, pausing at review gates to ask for your confirmation.
Limitation: Claude.ai cannot execute code or write files to your computer. For pipeline tasks that require script generation and execution (e.g. data cleaning), use Claude Code or Gemini CLI instead.
Running a skill with Gemini CLI

Gemini CLIGemini CLIGoogle's open-source agentic tool that runs in your terminal. Like Claude Code, it can read and write files and execute multi-step pipelines. Skill files from this site are compatible with both Claude Code and Gemini CLI. is Google's open-source agentic terminal tool. Skill files from this site are compatible with both Claude Code and Gemini CLI — the same file works with either.

1
Install Gemini CLI
npm install -g @google/gemini-cli
Then run gemini and authenticate with your Google account. Full docs at github.com/google-gemini/gemini-cli.
2
Download the skill file and set up your directory
Same as Claude Code — download the skill file and place it in a skills/ folder alongside your input files.
3
Run Gemini CLI with the skill
From your project directory:
cd my-analysis gemini > Read skills/thematic-analysis-skill.md and follow it.
Gemini CLI will read the skill file and execute the pipeline. The procedure is identical to Claude Code — the skill files are tool-agnostic.
4
Check the GEMINI.md conventions
Gemini CLI looks for a GEMINI.md file in your project root for persistent instructions. You can copy the skill file contents there if you want it to apply automatically to every session in that project.
Choosing between tools: Both work well. Claude Code tends to handle longer reasoning chains more reliably. Gemini CLI is free tier with a generous quota. Try both and use whichever fits your workflow.
